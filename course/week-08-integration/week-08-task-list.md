## Key Constraints for Week 8

### Must Use ONLY
From Weeks 1-7:
- All previous features (variables, I/O, types, conditions, loops, lists, strings, functions, dictionaries, files)

New in Week 8:
- assert statements for testing
- __name__ == "__main__" pattern
- Systematic error handling patterns
- Code organization principles
- Multi-function integration
- Professional documentation practices

### Must NOT Use
- Classes/objects - Week 12
- try/except - Week 10  
- Imports (except built-in) - Week 10
- Advanced testing frameworks - Beyond scope
- Complex error handling - Keep simple
- Version control systems - Conceptual only

## Success Criteria

- Students can organize code into logical functions
- Can implement comprehensive error handling
- Can test functions with assert statements
- Can build complete integrated applications
- Understand professional development practices
- Can debug complex multi-function programs
