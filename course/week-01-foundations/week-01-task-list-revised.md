## Key Differences from Original

1. **Philosophy**: Embrace AI rather than restrict it
2. **Focus**: Computational thinking over syntax
3. **Skills**: Prompt engineering as core competency
4. **Assessment**: Understanding and application over memorization
5. **Projects**: Document learning process, not just final code

## Success Criteria - REVISED

- Materials guide strategic AI use
- Computational thinking is emphasized throughout
- Students build mental models before coding
- Assessment measures understanding, not syntax knowledge
- AI is positioned as partner, not replacement
- Pattern recognition is developed systematically

## Dependencies

- Must test with multiple AI tools (Gemini, Claude, ChatGPT)
- Need real examples of AI overcomplication
- Require tested prompt progressions
- Mental model templates must be clear

## Notes for Development

- Every example should show the learning process
- Include "AI tried to do X but that's too complex" moments
- Emphasize the "why" before the "how"
- Make debugging a learning opportunity
- Show that understanding beats memorization
