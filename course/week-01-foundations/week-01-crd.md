# Week 1: Course Requirements Document (CRD)
## Python Foundations & AI Partnership

### Overview
Week 1 introduces students to Python programming WITH AI assistance, establishing the foundational mindset and practical skills for AI-enhanced learning. Students will learn to set up their development environment, understand basic syntax, and most importantly, develop a healthy relationship with AI tools as learning partners rather than solution providers.

### Learning Objectives
By the end of Week 1, students will be able to:

1. **Environment Setup & AI Integration**
   - Install and configure Python development environment
   - Set up and understand AI coding assistants (GitHub Copilot, Claude, ChatGPT)
   - Navigate between IDEs and AI interfaces effectively
   - Understand when and how to use AI appropriately

2. **Python Fundamentals**
   - Write and execute basic Python programs
   - Understand variables, data types, and type conversion
   - Use input() and print() functions effectively
   - Create programs that interact with users

3. **AI-Enhanced Learning**
   - Distinguish between using AI as a tutor vs. ghostwriter
   - Ask effective questions to AI for learning (not just solutions)
   - Verify AI-generated code and identify potential issues
   - Debug simple programs with AI assistance while maintaining understanding

4. **Problem-Solving Approach**
   - Break down problems into smaller steps
   - Write pseudocode before actual code
   - Test code incrementally
   - Document understanding with comments

### Common Pitfalls to Avoid

1. **AI Over-reliance**
   - Students copying without understanding
   - Accepting first AI suggestion without testing
   - Not reading error messages before asking AI
   - Using AI to write entire programs

2. **Complexity Creep**
   - AI suggesting advanced features too early (classes, decorators)
   - Students trying to add unnecessary features
   - Losing sight of core requirements
   - Adding error handling before understanding basics

3. **Google Colab Specific Issues**
   - Not saving notebooks regularly
   - Confusion between code and text cells
   - Runtime disconnections during long sessions
   - File persistence misunderstandings

4. **Conceptual Misunderstandings**
   - Thinking variables "contain" values (they reference them)
   - Confusing = (assignment) with == (comparison)
   - Not understanding input() always returns strings
   - Expecting Colab to remember variables between sessions

### Instructor Notes

#### Pacing Considerations
- Google Colab eliminates most setup issues
- Have backup Google accounts ready for access problems
- Pair struggling students with confident peers
- Keep solutions simple despite AI temptations
- Build in time for GitHub connection issues

#### AI Demonstration Scripts
Prepare examples showing:
- Good AI query: "Explain why my variable name '2cats' gives an error"
- Bad AI query: "Write a temperature converter program for me"
- AI overcomplication: Show how AI might suggest classes/functions too early

#### Support Strategies
- Create FAQ document for common setup issues
- Establish AI usage guidelines early and firmly
- Use peer mentors who've completed setup successfully
- Have "unplugged" activities ready for technical difficulties

#### Success Metrics
- 90% of students successfully run Python by end of Monday
- 80% can explain difference between int and string by Wednesday
- 100% complete at least basic version of temperature converter
- All students document at least one AI limitation discovered
